# 경쟁 서비스 비교 리서치 — 한글 → 영문 거래내역서 변환

조사 일자: 2026-04-26
조사 도구: Chrome DevTools MCP (실제 사이트 직접 접속 + 스냅샷 + 스크린샷)
대상: 한국어 거래내역(엑셀/CSV/PDF/이미지)을 영어로 변환하려는 사용자가 실제로 후보로 삼을 만한 범용 문서/이미지 번역 서비스 3곳

---

## 1) DeepL — Document Translation

- URL: https://www.deepl.com/ko/translator/files
- 스크린샷: `screenshot-deepl.png`

### 지원 입력
- PDF, Word (.docx), PowerPoint (.pptx)
- **Excel (.xlsx) 은 유료(Pro) 전용** — 무료 계정에서는 업로드 불가
- 별도 페이지에 "이미지 번역" 기능도 존재 (`/features/translate-image`)

### 가격 / 한도
- 무료 계정: **월 1~3개 문서** (페이지에 "월 1개" 표기 확인)
- Pro Starter / Advanced / Ultimate — 월 구독제, 외화 결제(USD/EUR) 위주, 한국 카드 결제 지원

### 강점
- 번역 품질이 가장 높다는 평이 일반적 (LLM 기반)
- 금융/법무/제약 산업 도입 레퍼런스 풍부 (KBC Bank, Deutsche Bahn 등)
- 원본 레이아웃·서식 보존 우수

### 약점 (= 우리 앱이 파고들 빈틈)
- **Excel은 유료 전용** → 한국 거래내역의 표준 양식(엑셀 다운로드)을 무료로 못 다룸
- **거래내역서 도메인 지식 없음** — 적요/종목/거래구분이 그냥 "텍스트"로 처리됨
- 컬럼명을 "Date / Withdrawal / Deposit / Balance" 같은 영문 표준 헤더로 정돈해주지 않음
- 미리보기 후 편집 불가, 제목·기간 같은 표지 정보 추가 불가
- 결제·저장·재열람 같은 워크플로우 자체가 없음 (1회성 번역기)

## 2) Naver Papago — 문서/이미지/텍스트 번역

- URL: https://papago.naver.com/?source=ko&target=en
- 스크린샷: `screenshot-papago.png`

### 지원 입력
- 4개 모드 탭: **텍스트 / 이미지 / 문서 / 웹사이트**
- 문서 모드: docx, pptx, xlsx, pdf 모두 지원 (DeepL 대비 Excel 무료 OK)
- 이미지 모드: OCR 내장 — 사진 속 한글 추출 후 번역

### 가격 / 한도
- 무료 (네이버 계정 로그인 권장)
- 텍스트 입력 3,000자 제한
- 문서는 용량/페이지 제한 명시 (페이지 내 표기에 따라 달라짐)

### 강점
- 한국어 특화 — 한글 고유명사, 지명, 일부 금융 용어 처리 우수
- 모든 문서 포맷을 무료로 받음
- 이미지 번역(OCR) 기본 탑재
- 네이버 생태계 사용자에게 친숙

### 약점 (= 우리 앱이 파고들 빈틈)
- **번역 품질이 도메인을 모름** — "적요"를 그냥 "Summary"로 옮기는 식의 직역 빈번
- 종목명·회사명에 일관성 없음 (같은 "삼성전자우"가 매 행마다 다르게 영문화될 수 있음)
- 표 구조(컬럼 헤더, 입금/출금/잔액)를 거래내역서 표준으로 재구성하지 않음
- 외국 고객 전달용 PDF 표지(제목·기간·발급자) 만들 수 없음
- 결제·소유권·이력 없음 — 재다운로드 시 다시 업로드해야 함

## 3) Google Translate — Documents Tab

- URL: https://translate.google.com/?sl=ko&tl=en&op=docs
- 스크린샷: `screenshot-google-translate.png`

### 지원 입력
- 페이지에 명시: **.docx, .pdf, .pptx, .xlsx**
- 텍스트/이미지/문서 3개 모드 탭 — 이미지 번역도 별도 모드 제공
- 엔진: Google Cloud Translation

### 가격 / 한도
- 무료 (구글 계정 선택)
- 100+ 언어 지원

### 강점
- 접근성 압도적 — 가입·결제 불필요
- 모든 주요 문서 포맷 무료
- 한국어→영어 일반 번역 품질 평균 이상

### 약점 (= 우리 앱이 파고들 빈틈)
- **표 레이아웃이 깨지는 사례가 빈번** (특히 PDF 거래내역서)
- 거래내역의 숫자 칼럼은 그대로 유지되지만, 헤더는 그냥 직역 ("거래일시" → "transaction date and time")
- 종목명/적요의 영문 표준화 없음 — 사람 이름은 로마자 변환이 일관되지 않음
- 산출물 후처리(제목/기간 추가, 외국 고객 전달용 PDF로 정돈) 불가
- 결제·저장·이력 기능 자체가 컨셉상 없음

## 종합 인사이트

**경쟁사가 못 메우는 빈틈 = 우리 앱의 포지셔닝**

1. **단발 사용자가 정확히 원하는 가격 모델**
   외국 고객에게 가끔 거래내역을 보내야 하는 자영업자/회계담당은 월 구독이 부담. "이번 한 번"에 페이지당 2,000원이 더 합리적.

2. **거래내역서 도메인 특화 번역**
   범용 번역기는 "적요/거래구분/잔액" 같은 컬럼명을 직역하지만, 우리는 표준 영문 헤더로 치환하고 종목명/회사명을 일관되게 영문화.

3. **외국 고객 전달용 후처리**
   제목, 기간, 명의자 정보가 들어간 깔끔한 PDF + Excel 동시 출력. DeepL/Papago/Google은 "번역만" 해주고 끝.

4. **보관·재열람 가치**
   결제한 거래내역은 계정에 영구 저장 — 외국 고객이 다시 요청해도 1초 안에 재발송 가능. 경쟁사는 매번 다시 업로드.

5. **이미지 거래내역서 처리**
   모바일 뱅킹 앱 캡처본을 그대로 업로드하면 Gemini Vision으로 추출 + 번역 + 표 형식 재구성. Google/Papago도 OCR은 되지만 표 형태로 정리 안 됨.

**리스크**
- 번역 품질은 Gemini API 의존 — 종목/적요 사전을 따로 누적해 일관성 보강 필요
- 공식 증빙용 아님(증명 도장·서식 X) → 마케팅 메시지에서 명확히 분리해야 신뢰 손상 안 됨
- Toss 테스트 키 노출 금지, 운영 시 실제 키는 Vercel 환경변수로만 관리
